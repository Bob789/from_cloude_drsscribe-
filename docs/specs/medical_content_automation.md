# איפיון מוצר: Medical Content Automation Platform (MCAP)

## 1. שם המוצר
**שם רשמי:** Medical Generate Content Engine (MGCE)  
**שם שיווקי:** Doctor Scribe Content Hub  
**גרסה:** 1.0.0 (Alpha)

---

## 2. תיאור כללי (Overview)
מערכת אוטומציה מבוססת AI ליצירה, ניהול והפצה של תוכן רפואי איכותי ודינמי. המערכת סורקת מקורות מידע מוסמכים (אוניברסיטאות, מכוני מחקר, סטארטאפים בתחום הבריאות), מתרגמת ומבצעת אדפטציה של התוכן לעברית, משלבת מדיה חזותית רלוונטית באופן אוטומטי, ומבצעת אופטימיזציה מלאה למנועי חיפוש (SEO). 

המערכת משתלבת באופן אינטגרלי עם פלטפורמת `parent-website` הקיימת (https://drsscribe.com/), ומזינה את הבלוג והקהילה בתוכן עדכני המעודד מעורבות גולשים והפניות לפורומים רפואיים.

---

## 3. מטרות עסקיות (Business Goals)
1. **חיסכון משאבים:** צמצום זמן יצירת תוכן ידני ב-80% באמצעות אוטומציה מלאה של מחקר, כתיבה ועריכה ראשונית.
2. **סמכות מקצועית (Authority):** ביסוס האתר כמקור ידע אמין ועדכני בתחום הרפואה הקונבנציונלית והמשלימה.
3. **גידול אורגני (SEO):** שיפור הדירוג במנועי חיפוש באמצעות תוכן עשיר במילות מפתח (Long-tail keywords), מובנה נכון טכנית ומתעדכן תדיר.
4. **מעורבות קהילה (Engagement):** הגדלת זמן שהייה באתר והנעת גולשים לדיונים בפורומים המקצועיים מתוך המאמרים (למשל: "קראת על הטיפול החדש? בוא להתייעץ בפורום אורתופדיה").

---

## 4. קהל היעד (Target Audience)
*   **מטופלים והקהל הרחב:** אנשים המחפשים מידע אמין על מצבים רפואיים, תזונה, בריאות הנפש וחידושים טכנולוגיים.
*   **אנשי מקצוע (HCPs):** רופאים ומטפלים המעוניינים להתעדכן במחקרים אחרונים ובמגמות בתחום ה-MedTech.
*   **סטודנטים לרפואה:** המחפשים סיכומים ומאמרים נגישים.

---

## 5. דרישות פונקציונליות (Functional Requirements)

### א. מנוע יצירת תוכן (Content Engine)
*   **Topic Generation:** יצירה אוטומטית של רעיונות לפוסטים על בסיס טרנדים (Google Trends), מילות מפתח מוגדרות מראש, וחדשות מאתרים סמכותיים (כגון PubMed, Nature, TechCrunch Health).
*   **Smart Drafting:** כתיבת מאמרים מלאים בעברית הכוללים:
    *   כותרת ראשית (H1) וכותרות משנה (H2, H3).
    *   פסקת פתיחה מסקרנת ("Hook").
    *   גוף המאמר מחולק לפסקאות קריאות.
    *   רשימות (Bullets/Numbers) לשיפור הקריאות.
    *   סיכום והנעה לפעולה (CTA).
*   **התאמת טון וסגנון:** יכולת הגדרה של "פרסונת הכותב" (למשל: "רופא מסביר", "יועץ תזונה", "חוקר אקדמי").
*   **אימות מקורות (Source Validation):** המערכת חייבת לצטט את המקור המקורי (Citation) ולקשר אליו, תוך הקפדה על דיוק רפואי (Fact-Checking לוגי בסיסי).

### ב. ניהול מדיה (Media Management)
*   **בחירה אוטומטית:** שליפת תמונות רלוונטיות ממאגרי Stock חינמיים (Unsplash, Pexels, Pixabay) באמצעות ניתוח סמנטי של הטקסט.
*   **אופטימיזציה:**
    *   דחיסת תמונות (WebP Format) לשיפור ביצועי טעינה.
    *   יצירה אוטומטית של טקסט חלופי (Alt Text) הכולל מילות מפתח לנגישות ו-SEO.

### ג. אופטימיזציה למנועי חיפוש (SEO Automation)
*   **Meta Data:** יצירה אוטומטית של Title Tag ו-Meta Description באורך האידיאלי.
*   **Keyword Injection:** שילוב טבעי של מילות מפתח ראשיות ומשניות בטקסט.
*   **Internal Linking:** זיהוי נושאים בטקסט וקישור אוטומטי למאמרים קיימים באתר או לדיונים רלוונטיים בפורום (למשל: אזכור "כאבי גב" יקשר לפורום אורתופדיה).
*   **Schema Markup:** הוספת סכמה מתאימה למאמרים רפואיים (MedicalWebPage / Article).

### ד. מנגנון אישור ובקרה (Editorial Workflow)
*   **דשבורד עורך:** ממשק ניהול המציג הצעות לפוסטים לפני פרסום ("Draft Mode").
*   **עריכה ידנית:** אפשרות לעורך אנושי לשנות טקסט, להחליף תמונה או לפסול פוסט לפני הפרסום הסופי.

## 6. ארכיטקטורה טכנית (Technical Architecture)

המערכת תפותח כ-Microservice עצמאי המתממשק ל-Backend הקיים ול-Frontend (Parent-Website).

### רכיבים טכנולוגיים (Tech Stack):
1.  **Backend Logic:** 
    *   שפה: Python (בשל ספריות ה-AI/NLP החזקות).
    *   Framework: FastAPI או התממשקות ל-Celery Workers הקיימים בפרויקט.
2.  **AI Models:** 
    *   תוכן: OpenAI GPT-4o / Claude 3.5 (באמצעות API).
    *   ראייה ממוחשבת (אופציונלי): CLIP לניתוח התאמת תמונות.
3.  **Integrations:**
    *   Unsplash API / Pexels API (תמונות).
    *   Google Trends API (זיהוי נושאים חמים).
4.  **Database:** שימוש ב-PostgreSQL הקיים (טבלאות חדשות: `generated_articles`, `content_sources`).
5.  **Frontend:** 
    *   דפי המאמרים: רכיבי React (Next.js) קיימים ב-`parent-website`.
    *   ממשק ניהול: הרחבה לפאנל הניהול הקיים.

### שילוב בקונפיגורציה הקיימת:
*   **Docker:** יצירת קונטיינר חדש `content-generator` ב-`docker-compose.yml`.
*   **Scheduling:** שימוש ב-Celery Beat לתזמון משימות (למשל: "יצירת 3 מאמרים חדשים כל בוקר ב-08:00").
*   **Security:** אימות מול ה-API Gateway הקיים למניעת הזרקת תוכן זדוני.

## 7. תרחיש שימוש לדוגמה (User Story)

1.  **טריגר:** המערכת מזהה טרנד עולה ב-Google Trends: "טיפולים חדשים למיגרנה".
2.  **מחקר:** המערכת סורקת אתרי חדשות רפואיים (כגון MedicalNewsToday) ומוצאת מאמר עדכני על תרופה חדשה שאושרה הבוקר.
3.  **עיבוד:** ה-AI מסכם את המאמר, מתרגם לעברית, ומנסח אותו כפוסט בבלוג תחת הכותרת: "בשורה לסובלים ממיגרנה: הכירו את הטיפול החדש".
4.  **העשרה:** המערכת מושכת תמונה של אישה מחזיקה את ראשה (מתוך Unsplash) ומוסיפה Alt Text: "טיפול במיגרנה וכאבי ראש".
5.  **קישוריות:** המערכת מוסיפה בסוף המאמר הזמנה: "סובלים ממיגרנות? שתפו אותנו בפורום נוירולוגיה".
6.  **פרסום:** המאמר נשמר כ-Draft. מנהל התוכן מקבל התראה, מאשר את התוכן בלחיצת כפתור, והמאמר עולה לאוויר ב-drsscribe.com.
