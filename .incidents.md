## INC-001 | 2026-03 | Backend | `/visits/patient/4` returns 500
**Symptom:** `MultipleResultsFound` exception on visit summary endpoint.
**Root cause:** Duplicate rows in `summaries` table for the same `visit_id`. Code used `scalar_one_or_none()` which throws when >1 result exists.
**Fix:** Changed to `order_by(Summary.created_at.desc()).scalars().first()` to always get the latest.
**Lesson:** Never use `scalar_one_or_none()` on columns that don't have a UNIQUE constraint. Always use `.first()` with `order_by` when duplicates are possible.

---

## INC-002 | 2026-03 | Backend | Google Calendar OAuth `redirect_uri_mismatch`
**Symptom:** 400 error when connecting Google Calendar. `redirect_uri_mismatch`.
**Root cause:** `client_secret.json` inside the Docker container had an old client ID (from a different GCP project). The file on disk was updated but the container was started with `--no-build`, so the old file persisted in the image.
**Fix:** `docker compose build backend` then restart.
**Lesson:** NEVER use `--no-build` after changing files that are COPY'd into the Docker image. Always rebuild.

---

## INC-003 | 2026-03 | Backend | Google OAuth `access_denied` (403)
**Symptom:** After fixing redirect_uri, Google login shows "Access blocked" screen.
**Root cause:** OAuth consent screen was in "Testing" mode in Google Cloud Console, limiting access to test users only.
**Fix:** Published the app in Google Cloud Console → APIs & Services → OAuth consent screen → Publish.
**Lesson:** Google OAuth consent screen must be in "Production" mode for external users. Testing mode = whitelist only.

---

## INC-004 | 2026-03 | Frontend | Flutter build fails after branding rename
**Symptom:** Dart compilation error — invalid class names with spaces.
**Root cause:** `sed` replaced ALL occurrences of "MedScribe" including Dart class identifiers. `MedScribeApp` became `Doctor Scribe AIApp` (spaces in class names are illegal in Dart).
**Fix:** Reverted class identifiers: `Doctor Scribe AIApp` → `MedScribeApp`, kept display strings as "Doctor Scribe AI".
**Lesson:** When doing global text replacement, NEVER replace inside code identifiers (class names, function names, variable names). Only replace display strings, labels, and comments.

---

## INC-005 | 2026-03 | Frontend | "Doctor Scribe AI AI" double text
**Symptom:** UI showed "Doctor Scribe AI AI" instead of "Doctor Scribe AI".
**Root cause:** Original text was "MedScribe AI". Replacing "MedScribe" → "Doctor Scribe AI" turned it into "Doctor Scribe AI AI".
**Fix:** `sed -i 's/Doctor Scribe AI AI/Doctor Scribe AI/g'` across all files.
**Lesson:** When replacing brand names, check for existing suffixes (AI, Pro, etc.) that would create duplicates. Do a post-replace search for double words.

## INC-006 | 2026-03 | Parent Website | Next.js build fails after rename
**Symptom:** TSX compilation error — function names with spaces.
**Root cause:** Same as INC-004. `sed` replaced "MedScribe" in TSX identifiers like `AboutMedScribePage` → `AboutDoctor Scribe AIPage`.
**Fix:** Reverted TSX identifiers while keeping display strings.
**Lesson:** Same as INC-004 — applies to ALL languages, not just Dart.

## INC-007 | 2026-03 | Parent Website | Sticky header not working
**Symptom:** Header scrolls away with page content instead of staying fixed.
**Root cause:** `html { overflow-x: hidden }` in CSS breaks `position: sticky` in all browsers.
**Fix:** Changed to `html { overflow-x: clip }` which prevents horizontal overflow without breaking sticky.
**Lesson:** `overflow: hidden` on html/body breaks `position: sticky`. Always use `overflow: clip` instead.

---

## INC-008 | 2026-03 | Backend | Container has stale files after code change
**Symptom:** Changes to backend Python files not taking effect after restart.
**Root cause:** Container was restarted with `docker compose up -d` without rebuilding. The Python files are COPY'd into the image at build time, so restart just reuses the old image.
**Fix:** Always run `docker compose build <service>` before `up -d` when code files changed.
**Lesson:** Docker containers don't auto-update from disk. Code changes require `build` → `up -d`. Only volume-mounted files update without rebuild.

## INC-009 | 2026-03-18 | Backend | Celery asyncio event loop crash
**Symptom:** All transcription/summary tasks crash with "Future attached to a different loop".
**Root cause:** Global SQLAlchemy async engine bound to import-time event loop. Celery's `run_async()` creates new loop per task → conflict.
**Fix:** Create fresh engine+session per task via `_make_session()`, dispose after use.
**Lesson:** Never share async DB engines across event loops in Celery. Each task needs its own engine.

---

## INC-010 | 2026-03-18 | Backend | Celery worker runs old code after backend rebuild
**Symptom:** Fixed tasks.py but celery-worker kept failing the same way.
**Root cause:** `celery-worker` builds a SEPARATE Docker image from `backend`. `build backend` does NOT update celery.
**Fix:** Always `build backend celery-worker` together + `--force-recreate`. Verify with import check.
**Lesson:** ALWAYS build both backend AND celery-worker together. ALWAYS verify new code is loaded.

---

## INC-011 | 2026-03-18 | Backend | Empty AI summaries (status=done, 0 chars)
**Symptom:** Summary task completes successfully but all text fields are empty.
**Root cause:** Transcription is encrypted in DB (AES-256). Summary task read encrypted base64 text, sent it to LLM, LLM returned empty/garbage. Missing `decrypt_transcription_fields()` before processing.
**Fix:** Added decrypt call before mask_pii/summarize in tasks.py.
**Lesson:** When data is encrypted at rest, ALWAYS decrypt before processing. Check full pipeline: read → decrypt → process → encrypt → save.

---

## INC-012 | 2026-03-18 | Frontend | Times shown in UTC instead of local
**Symptom:** Visit at 15:10 Israel time shows as 13:10 in the app.
**Root cause:** `DateTime.parse(dateStr)` without `.toLocal()` in Flutter date formatters.
**Fix:** Added `.toLocal()` to all formatters (visit_card, search_result_card, activity/audit screens).
**Lesson:** DB stores UTC (correct). Frontend MUST call `.toLocal()` before displaying dates.

---

## INC-013 | 2026-03-18 | Parent Website | Product page CSS disappeared
**Symptom:** drsscribe.com/product page lost all styling — plain HTML.
**Root cause:** globals.css was rewritten for homepage redesign, removing all product/articles/forum CSS.
**Fix:** Merged both CSS sets into globals.css.
**Lesson:** When editing shared CSS, NEVER overwrite — append or merge. Check which pages use the file.
